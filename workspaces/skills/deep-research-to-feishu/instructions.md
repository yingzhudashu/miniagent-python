# Deep Research to Feishu Doc

## 目标
针对用户指定的主题进行全网深度调研，生成结构化、专业级的 Markdown 格式报告，并通过飞书云文档交付。交付时**必须**提供可直接点击的通用飞书域名链接。

## 标准执行流

### 1. 深度信息搜集
- 使用 `web_search` 获取主题的广度信息（官方资料、GitHub、技术文档、社区讨论等）。
- 使用 `fetch_url` 或 `browser_extract_text` 深入提取核心链接的详细技术内容。
- 搜集范围应涵盖：项目背景、核心功能、技术架构、应用场景、优缺点分析、生态与社区现状。

### 2. 结构化报告撰写
将搜集的信息整理为完整的 Markdown 报告，建议包含以下 8 大模块：
1. **项目概述**：定义、定位、核心数据（Star 数、活跃度等）。
2. **核心功能特性**：使用表格列出关键特性与说明。
3. **技术架构深度解析**：技术栈、组件图、数据流、核心机制（如内存管理、安全模型）。
4. **典型应用场景**：列举 4-6 个落地场景。
5. **优势与挑战（优缺点对比）**：使用 Pros/Cons 表格对比。
6. **生态系统与社区现状**：衍生项目、商业化、安全规范等。
7. **未来展望**：技术演进与行业趋势预测。
8. **总结**：一句话核心价值与部署建议。

### 3. 飞书文档交付
- 调用 `feishu_doc`，`action=create` 创建新文档，标题建议为 `《{主题} 深度调研报告》`。
- 调用 `feishu_doc`，`action=append`（或 `write`）将报告内容写入文档。
  - ⚠️ **注意**：若报告过长，请**按章节拆分**多次调用追加，防止单次截断。
- 写入完成后，向用户返回确认信息。

## 关键约束与最佳实践 (Best Practices)
- 🔗 **链接交付规范**：最终回复中**必须**包含通用格式的完整链接：
  `https://feishu.cn/docx/{document_id}`
  🚫 **禁止**仅返回 Document ID 或要求用户自行拼接域名。
- ✅ **完整性确认**：交付时明确告知用户“文档已创建，内容已完整写入”。
- 🛡️ **防坑指南**：
  - 长文本写入务必分段，监控 Append 操作的返回状态。
  - 调研内容需保持客观、详实，避免空洞的泛泛而谈。
