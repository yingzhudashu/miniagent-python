# 架构文档

> Mini Agent Python 系统架构说明

## 1. 系统概览

Mini Agent 是一个基于 LLM 的智能个人助手，采用 **Plan-then-Execute** 两阶段架构。

```
用户输入 → CLI/飞书 → Agent编排 → [规划器 → 执行器] → 工具调用 → 回复
                                    ↓
                              自我优化子系统
```

## 2. 核心架构

### 2.1 Agent 身份系统

Agent 在运行时有明确的身份认知，通过 System Prompt 注入给 LLM：

| 角色 | 身份 | System Prompt |
|------|------|---------------|
| 规划器 | `MiniAgent 的规划器` | "你是 MiniAgent 的规划器。你是一个任务规划专家..." |
| 执行器 | `MiniAgent` | "你是 MiniAgent，一个基于 Python 的轻量级 LLM Agent..." |

身份定义在 `src/core/executor.py` 和 `src/core/planner.py` 中，通过 `AGENT_NAME` 和 `AGENT_IDENTITY` 常量导出。

### 2.2 两阶段引擎

**Phase 1 - 规划器 (Planner)**:
- 接收用户输入，调用 LLM 生成结构化计划 (StructuredPlan)
- 决定需要调用哪些工具、按什么顺序
- 3 次重试机制，全部失败时降级为 fallback 简单计划

**Phase 2 - 执行器 (Executor)**:
- ReAct 循环执行计划中的工具调用
- 循环检测、记忆注入、上下文管理
- 支持 `on_tool_call` 回调（用于 CLI 实时进度显示）

**Agent 层**:
- 编排规划器和执行器
- 支持跳过规划直接执行 (`.plan` 命令)
- 管理对话历史、工具监控和记忆

### 2.3 工具系统

工具按功能分组到工具箱 (Toolbox)：

| 工具箱 | 工具 | 说明 |
|--------|------|------|
| filesystem | read_file, write_file, edit, list_files | 文件操作 |
| exec | run_command | 命令执行（含子进程跟踪） |
| web | web_fetch, web_search | 网页工具 |
| skills | search_skills, install_skill, list_skills | 技能管理 |
| self_opt | .optimize inspect/status/auto | 自我优化 |

**子进程管理**: 通过 `process_tracker.py` 跟踪所有子进程 PID，退出时自动清理孤儿进程。

### 2.4 技能系统

技能是可插拔的代码包，位于 `skills/` 目录：

```
skills/
├── default/     # 默认技能
│   ├── SKILL.md # 技能描述和配置
│   └── tools/   # 技能附加工具
└── custom/      # 自定义技能
```

**加载流程**:
1. 启动时扫描 `skills/` 目录
2. 解析每个技能包的 `SKILL.md`
3. 注册技能贡献的工具到全局注册表
4. 注入技能系统提示到 Agent

**ClawHub**: 支持从 ClawHub.ai 搜索、下载、安装远程技能。

### 2.5 记忆系统

三层记忆架构：

```
┌─────────────────┐
│  上下文记忆     │ ← 当前对话窗口（最短期）
├─────────────────┤
│  会话记忆       │ ← 当前会话对话历史
├─────────────────┤
│  语义记忆       │ ← 关键词索引的长期记忆
└─────────────────┘
```

- **上下文记忆**: 当前对话的短期上下文，随对话窗口滑动
- **会话记忆**: 会话级别的对话历史，跨轮次保留
- **语义记忆**: 通过关键词索引搜索的长期记忆

### 2.6 会话管理

每个会话独立隔离：
- 独立的工具注册表
- 独立的工作空间目录
- 独立的对话历史
- 支持会话间的工具升降维 (promote/demote)

## 3. 集成层

### 3.1 CLI 交互

```
python -m src → 交互式循环
  ├── DisplayManager    → 终端显示管理（去重、颜色、spinner）
  ├── .stats            → 工具使用统计
  ├── .skills           → 已加载技能
  ├── .sessions         → 会话列表
  ├── .profile          → 切换模型预设
  ├── .optimize         → 自我优化
  ├── .plan             → 跳过规划直接执行
  └── quit              → 退出
```

**DisplayManager**:
- 去重输出：相同内容不重复打印
- 历史稳定：新输出不影响历史显示
- 实时进度：LLM 思考时显示 spinner
- 工具回调：`on_tool_call` 实时显示工具执行结果
- 彩色输出：ANSI 转义码区分信息/警告/错误

### 3.2 飞书集成

**两种模式**:

| 模式 | 文件 | 特点 |
|------|------|------|
| 长轮询 | `poll_server.py` | WebSocket 连接飞书服务器，无需公网 IP |
| Webhook | `server.py` | HTTP 服务器接收事件推送，需要公网可达 |

**处理流程**:
```
飞书事件 → 去重检查 → 顺序队列 → Agent 处理 → 回复
```

**安全机制**:
- 内存 + 磁盘双重去重
- 同一聊天室顺序队列（防并发）
- 消息防抖（1.5s 内合并）
- 单客户端单例

## 4. 自我优化子系统

### 4.1 组件

```
┌─────────────┐     ┌──────────────┐     ┌────────────────┐
│  Inspector  │────→│  Proposal    │────→│  Test Runner   │
│  自我审视   │     │  Engine      │     │  测试执行器    │
└─────────────┘     │  提案引擎    │     └────────────────┘
                    └──────────────┘              │
┌─────────────┐     ┌──────────────┐              │
│  Researcher │────→│  Learner     │              │
│  外部调研   │     │  学习器      │              │
└─────────────┘     └──────────────┘              │
                                                   ▼
┌─────────────┐     ┌──────────────┐     ┌────────────────┐
│  Metrics    │←────│  Auto-       │←────│  Git Snapshot  │
│  指标仪表盘 │     │  Optimizer   │     │  快照管理      │
└─────────────┘     │  编排器      │     └────────────────┘
                    └──────────────┘
```

### 4.2 自动优化流程

```
1. 自我审视 (Inspector) → 分析代码质量、架构完整性、痛点
2. 外部调研 (Researcher) → 搜索最佳实践和架构模式
3. 提案生成 (ProposalEngine) → 基于模板匹配生成优化方案
4. Git 快照 → 创建快照用于回滚
5. 执行优化 → 应用文件变更，运行测试
6. 自动修复 → 测试失败时 LLM 生成修复补丁（最多 2 次）
7. 回滚 → 修复失败时回滚到 Git 快照
8. 记录日志 → 写入 OPTIMIZATION_LOG.md
```

### 4.3 风险分级

| 风险等级 | 执行策略 | 示例 |
|----------|----------|------|
| low | 自动执行 | 添加新工具、改进注释 |
| medium | 用户确认 | 修改工具逻辑、添加依赖 |
| high | 必须确认+快照 | 修改核心模块 |
| destructive | 必须确认+手动审核 | 删除文件 |

## 5. 安全

### 5.1 路径沙箱

所有文件操作必须通过沙箱验证：
1. 解析输入路径为绝对路径
2. 检查是否在允许的目录范围内
3. 符号链接解析为真实路径
4. 阻止目录穿越攻击

### 5.2 单实例管理

确保同一时间只有一个 Agent 实例运行：
- PID 文件锁机制
- 进程存活检测
- 支持强制启动和停止

### 5.3 子进程跟踪

`process_tracker.py` 提供：
- 自动记录所有 `run_command` 产生的子进程 PID
- 启动时检测孤儿进程并清理
- 退出时优雅终止所有活跃子进程

## 6. 开发

### 6.1 添加新工具

1. 在 `src/tools/` 创建工具文件
2. 在 `__init__.py` 导出
3. 注册到工具箱

### 6.2 添加新技能

1. 在 `skills/` 创建技能目录
2. 编写 `SKILL.md`
3. 实现 `tools.py`（可选）
4. 重启 Agent 自动加载

### 6.3 运行测试

```bash
pytest tests/ -v
pytest tests/test_sandbox.py -v
pytest tests/test_integration.py -v
```

### 6.4 日志规范

- 库代码：使用 `get_logger(__name__)` 而非 `print()`
- CLI 代码：使用 `DisplayManager` 方法输出
- 结构化日志：使用 `append_log()` 写入 JSONL 文件
